# 機能追加仕様 — お気に入り & 学習進捗（Opus 4.8 設計）

閲覧者のブラウザに状態を保存し、(A) カードの「お気に入り」登録、(B) カードを「学習済み」にチェック、(C) 学習進捗をかっこよく可視化する。ログイン不要・端末ローカルで完結。

## 0. 永続化の方式

`localStorage` を使う（要件の「クッキー機能」と同じ目的＝この端末に記憶。Cookieはサーバ送信され容量も4KBと小さく不適のため、クライアント永続化の標準である localStorage を採用）。キー：

- `ai-navi:favorites` … お気に入りカードIDの配列（JSON）
- `ai-navi:completed` … 学習済みカードIDの配列（JSON）
- `theme` … 既存（ダークモード）

不正値・破損時は空配列にフォールバックすること。

## 1. 状態管理フック

`src/hooks/useCardProgress.js`（単一フックでまとめてよい）:
```
const {
  favorites,        // Set<string>
  completed,        // Set<string>
  isFavorite(id), isCompleted(id),
  toggleFavorite(id), toggleCompleted(id),
  resetAll(),       // 進捗・お気に入りを全消去（確認ダイアログ付き）
} = useCardProgress();
```
- 内部で `useState(()=>load())` 初期化、変更時に localStorage へ保存（useEffect）。
- 複数タブ同期は任意（`storage` イベント購読があれば尚良い、必須ではない）。

App.jsx でこのフックを1回呼び、各コンポーネントに props/Context で配る（軽いので props バケツリレーでよい。煩雑なら React Context を1つ作る）。

## 2. カード（CardItem）への追加

- 右上に **ハートのお気に入りトグル**（♡ / ♥）。クリックで toggleFavorite。カード本体のモーダル展開とイベントが競合しないよう `stopPropagation`。お気に入り時はブランドアクセント色 or ローズ系で塗り、軽くポップするアニメ。aria-pressed / aria-label 付与。
- 学習済みのとき、カードに**達成スタイル**：左上または上帯に「✓ 学習済み」バッジ（緑系）、カードをわずかに沈める（opacity 0.92＋緑のリング `ring-2 ring-green-400/40`）。やりすぎず上品に。
- カード上のメタ行に小さな done チェック（丸＋✓）を置き、クリックで toggleCompleted も可（任意・モーダル内でも可なら省略可）。

## 3. モーダル（CardModal）への追加

フッターのCTA行に2つのアクションを追加：
- 「♥ お気に入り」トグルボタン（状態で塗り変化）
- 「✓ 学習済みにする」/「学習済み（解除）」トグルボタン（完了時は緑塗り＋チェック）。押すと気持ちよいマイクロインタラクション（軽いスケール）。
既存の「公式サイトへ」CTAと並べる。モバイルでは縦積み。

## 4. 学習進捗ダッシュボード（最重要・"かっこよく"）

新コンポーネント `src/components/ProgressDashboard.jsx`。配置：Hero と ProviderSpotlight の間（または ProviderSpotlight 直後、ギャラリーの直前）。常に見える位置で、学習の動機づけになるバンド。

構成：
1. **円形プログレスリング（SVG）**：全体の学習率 %（completed/total*100、四捨五入）。
   - 二重円。背景トラック（薄いグレー）＋前景アーク。
   - 前景は**グラデーションのストローク**（3社カラーを混ぜた conic/linear グラデを `<linearGradient>` で定義）。
   - `stroke-dasharray`/`stroke-dashoffset` で値を表現し、`transition` で滑らかに伸びる。
   - 中央に大きな「31%」、その下に「12 / 39 学習済み」。
   - 数値は 0→現在値へ**カウントアップ**するアニメ（簡易でよい：requestAnimationFrame か CSS）。
2. **カテゴリ別 or プロバイダ別の内訳バー**：各プロバイダ（共通・ChatGPT・Gemini・Claude）について「3 / 12」と、ブランドカラーの**横バー**（width をアニメ）。4本並べる。
3. **お気に入り数の表示**：「♥ 5 件のお気に入り」チップ。クリックでお気に入りフィルタへ。
4. **マイルストーン文言**：進捗に応じて励ましを出す。
   - 0%：「最初のカードを学んでみよう 🌱」
   - 1–40%：「いいスタート！この調子 🚀」
   - 41–80%：「順調！折り返し地点を突破 💪」
   - 81–99%：「ゴールはすぐそこ ✨」
   - 100%：「全カード制覇！おめでとう 🎉」＋ささやかな祝福演出（リングが輝く/絵文字）。
5. **リセット**：小さく「進捗をリセット」リンク（確認の上 resetAll）。

デザイン：カード型のパネル（rounded-3xl、やわらかい影、ライト/ダーク対応）。左にリング、右に内訳という2カラム（モバイルは縦積み）。背景に3社グラデのほのかな装飾。

## 5. ヘッダーへのミニ進捗（任意だが推奨）

ヘッダー右側（ダークトグルの近く）に**小さな進捗リング or 「学習 31%」チップ**を常時表示。クリックでダッシュボードへスムーズスクロール。モバイルでは折りたたみ可。

## 6. フィルタ連携（FilterBar）

クイックトグルを追加（既存のプロバイダ/カテゴリ/検索とAND）：
- 「♥ お気に入り」… favorites のみ表示
- 「未学習」… completed でないもののみ
- （任意）「学習済み」
トグルはオン/オフのチップ。0件時は既存の空状態（メッセージとリセット）。

## 7. アニメーション/品質

- リングのアーク、内訳バーの width、カウントアップは滑らかに（200–800ms ease）。
- お気に入り/完了トグルは触ると気持ちいい（scale/色変化）。過度な点滅は避ける。
- アクセシビリティ：トグルは button、aria-pressed、十分なコントラスト、キーボード操作可。
- 既存の `content.js` は引き続き編集禁止対象（※ただし動画ID追加は別途Opusが行う。実装側は触らない）。
- ダークモード・モバイルでも崩れないこと。

## 8. 完了条件

- `npm run build` がクリーンに通る。
- お気に入り/学習済みがリロード後も保持される（localStorage）。
- ダッシュボードの % ・内訳・お気に入り数が状態と常に一致し、トグルで即時更新＆アニメ。
- お気に入り/未学習フィルタが機能。
- README に新機能（永続化の仕組み＝localStorage、リセット方法）を追記。
